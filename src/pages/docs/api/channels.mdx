# Channels API

Channels connect ainative to external messaging platforms. Each channel configuration stores credentials and connection details for a Slack workspace, Telegram bot, or generic webhook. Inbound endpoints receive messages from these platforms and route them through the gateway for agent processing.

## Quick Start

Configure a Slack channel, test the connection, and send a message — a typical integration flow:

<CodeExample ts={`// 1. Create a Slack channel configuration
const createRes: Response = await fetch('/api/channels', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    name: 'Engineering Alerts',
    channelType: 'slack',
    config: {
      webhookUrl: 'https://hooks.slack.com/services/T0001/B0001/xxxx',
    },
  }),
});
const channel: { id: string; channelType: string; testStatus: string } = await createRes.json();
// -> { id: "ch-3a4b-5c6d", channelType: "slack", testStatus: "untested", ... }

// 2. Test the connection — sends a test message through the adapter
const testRes: Response = await fetch(\`/api/channels/\${channel.id}/test\`, {
  method: 'POST',
});
const test: { testStatus: string } = await testRes.json();
console.log(\`Test: \${test.testStatus}\`); // "ok" or "failed"

// 3. Enable bidirectional messaging (receive Slack events too)
await fetch(\`/api/channels/\${channel.id}\`, {
  method: 'PATCH',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ direction: 'bidirectional' }),
});

// 4. List all configured channels
const listRes: Response = await fetch('/api/channels');
const channels: Array<{ name: string; channelType: string; status: string; testStatus: string }> = await listRes.json();
channels.forEach((ch) => {
  console.log(\`\${ch.name} (\${ch.channelType}) — \${ch.status} [\${ch.testStatus}]\`);
});`} python={`import requests

# 1. Create a Slack channel configuration
create_res = requests.post(
    "http://localhost:3000/api/channels",
    json={
        "name": "Engineering Alerts",
        "channelType": "slack",
        "config": {
            "webhookUrl": "https://hooks.slack.com/services/T0001/B0001/xxxx",
        },
    },
)
channel: dict = create_res.json()
# -> { "id": "ch-3a4b-5c6d", "channelType": "slack", "testStatus": "untested", ... }

# 2. Test the connection — sends a test message through the adapter

test: dict = test_res.json()
print(f"Test: {test['testStatus']}")  # "ok" or "failed"

    f"http://localhost:3000/api/channels/{channel['id']}",

channels: list[dict] = list_res.json()
for ch in channels:
    print(f"{ch['name']} ({ch['channelType']}) — {ch['status']} [{ch['testStatus']}]")`} />

### Create Channel

<SchemaTable title="Request Body" fields={[

### Test Channel Connection

<EndpointCard method="POST" path="/api/channels/{id}/test" description="Test connectivity to a channel by sending a test message through its adapter. Updates the channel's testStatus field.">

<SchemaTable title="Response Body" fields={[

  { name: 'error', type: 'string', required: false, description: 'Error message if test failed' },

Send a test message to verify the channel is configured correctly — always test after creating or updating config:

<CodeExample ts={`// Test connectivity — sends a test message through the adapter

  method: 'POST',

  console.log('Channel is working — test message sent');

  console.error(\`Test failed: \${error}\`);
}`} python={`import requests

    print("Channel is working — test message sent")
else:
    print(f"Test failed: {data.get('error')}")`} />

Example response (success):

```json

  "testStatus": "ok"
}
```

Example response (failure):

```json

  "error": "Webhook returned 403 — check the webhook URL is still valid"
}
```

**Errors:** `404` — Channel not found, `500` — Adapter error

</EndpointCard>

## Inbound Webhooks

### Slack Inbound

<EndpointCard method="POST" path="/api/channels/inbound/slack?configId={id}" description="Receives Slack Events API callbacks. Handles url_verification challenges and event_callback messages. Verifies request signatures using the channel's signingSecret. Responds within 3 seconds per Slack requirements.">

<ParamTable title="Query Parameters" params={[
  { name: 'configId', type: 'string', required: true, description: 'Channel config ID to route the message to' },
]} />

**Response** `200` — `{ "ok": true }` (or challenge response)

**Errors:** `400` — Missing configId or invalid JSON, `401` — Missing signingSecret, `403` — Invalid signature, `404` — Channel not found

</EndpointCard>

### Telegram Inbound

<EndpointCard method="POST" path="/api/channels/inbound/telegram?configId={id}&secret={token}" description="Receives Telegram Bot API webhook updates. Verifies the shared secret token. Bot messages are silently skipped to prevent loops.">

<ParamTable title="Query Parameters" params={[
  { name: 'configId', type: 'string', required: true, description: 'Channel config ID to route the message to' },
  { name: 'secret', type: 'string', required: true, description: 'Shared webhook secret for verification' },
]} />

**Response** `200` — `{ "ok": true }`

**Errors:** `400` — Missing configId, `401` — Missing webhookSecret in config, `403` — Invalid secret, `404` — Channel not found

</EndpointCard>

### Telegram Poll (Internal)

## Channel Types

| Type | Required Config | Direction |
|------|----------------|-----------|
| **slack** | `webhookUrl`, `signingSecret` (for inbound) | Outbound or bidirectional |
| **telegram** | `botToken`, `chatId`, `webhookSecret` (for inbound) | Outbound or bidirectional |
| **webhook** | `url` | Outbound only |
