## First-sync mode

On the first run of `apply-product-docs`, no page has `{/* SYNC:START */}` / `{/* SYNC:END */}` markers. The skill detects this and enters **first-sync mode**:

1. For each page in "Direct mappings", read the current website body.
2. Propose a diff showing where SYNC markers would be inserted (typically right after the hero `<BrowserFrame>` and closing before any website-only chrome).
3. Surface the diff to the user per-page for confirmation â€” website authors may want to adjust boundary placement (e.g., "keep my custom `## Deployment Notes` section outside the markers").
4. Only after confirmation does Execute Phase write the markers and regenerate body content.

Subsequent runs operate in incremental mode: content inside markers is replaced; content outside is preserved.
