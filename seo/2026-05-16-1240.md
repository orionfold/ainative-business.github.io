## GSC Drilldown — Example URLs per Reason (added 2026-05-16 post-investigation)

Investigation triggered by user note: "GSC has most issues. investigate why it did not work drill-down."

**Root cause of original drilldown failure (Plan Phase):** Each reason row in the Pages overview is a `<tr class="nJ0sOc" jsaction="click:AJI9fd;">`. Synthesizing a click event via `dispatchEvent(new MouseEvent('click', { bubbles: true }))` pushes `location.href = …/index/drilldown?item_key=<key>` but does **NOT** trigger the SPA's view re-render — the table continues to show the parent overview. Google's jsaction handler appears to require a real user-input event (or a state-machine bootstrap) before re-rendering.

**Working pattern:** synthetic-click harvests the opaque `item_key` per reason; then `navigate()` directly to the drilldown URL forces a fresh SPA bootstrap.
